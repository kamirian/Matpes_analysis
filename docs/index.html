Why Average Error Is Not Enough

Current FP evaluations overwhelmingly rely on a single number — the MAE or RMSE of force magnitude. This is convenient but incomplete. A model with low average error can still fail in ways that matter enormously in practice:

The problem: A model can predict forces that point in completely wrong directions, or produce catastrophically large errors on a small fraction of atoms — and both failure modes are masked by the average.

These failures have real consequences. Static relaxations and NEB calculations typically require force convergence of 0.001–0.01 eV/Å. Misdirected or catastrophically large force predictions during molecular dynamics can destabilize trajectories and produce unphysical atomic dynamics.

Metric Definitions

This benchmark introduces two complementary per-atom metrics and evaluates them across three physically motivated force regimes.

|ΔF|
Force magnitude error — eV/Å
|ΔF| = | |F_FP| − |F_DFT| |
How far off is the predicted force strength? Drives errors in bond lengths, lattice constants, and energy barriers. MAE and RMSE of |ΔF| are reported; RMSE is sensitive to catastrophic outliers.
|Δθ|
Force direction error — degrees
|Δθ| = arccos( F_FP · F_DFT / (|F_FP| |F_DFT|) )
How wrong is the predicted force direction? A large angular error pushes atoms toward wrong energy minima even when the magnitude looks acceptable. Evaluated only for atoms where |FDFT| > 0.01 eV/Å to avoid division noise.

Fraction metrics

frac(<δ & <α) ↑
DFT-substitutable fraction — higher is better
% atoms: |ΔF| < δ AND |Δθ| < α
Joint criterion requiring both low magnitude error and accurate direction. Reported for (δ, α) = (0.01 eV/Å, 1°) and (0.01 eV/Å, 20°). This is the fraction of atoms where the FP force is precise enough to substitute DFT.
frac(>1 eV/Å) ↓
Catastrophic error fraction — lower is better
% atoms: |ΔF| > 1 eV/Å
Even a small fraction of atoms with |ΔF| > 1 eV/Å can crash an MD simulation or corrupt a relaxation trajectory. This metric is invisible to average error but highly consequential in practice.

Three analysis regimes

DFT-substitutable

Highly accurate predictions

|ΔF| < 0.01–0.1 eV/Å and |Δθ| < 1–20°. Required for converged relaxations and reliable NEB barriers. For most FPs, jointly satisfying both criteria is rare (<20% of atoms).

Catastrophic failures

Large-error atoms

|ΔF| > 1 eV/Å. Rarely reported by average metrics, but highly consequential — even a small fraction of these can crash an MD simulation or corrupt a trajectory.

Far-from-equilibrium

High-force configurations

|FDFT| ≥ 1 eV/Å. About 26.9% of MatPES-PBE atoms fall here vs only ~6% in MPtrj. Most benchmarks under-test this regime that is critical for MD stability.

Key finding: The fraction of low-error atoms and the fraction of catastrophic-error atoms are more informative than MAE/RMSE alone. The test dataset must include a substantial fraction of far-from-equilibrium atoms to meaningfully evaluate FP reliability beyond near-equilibrium relaxation.

How to Submit Your Model

We welcome evaluation of new foundation potentials on these benchmarks. Follow the steps below to run the same evaluation pipeline and report results using the same metrics and dataset splits.

Reproducibility: All analysis code is open-source. By following this protocol, your results will be directly comparable to the values in the leaderboard above.

Set up the environment

Clone the repository and install dependencies. Each FP has its own environment requirements — see the README for per-model installation instructions (ASE, MACE, CHGNet, MatGL, UMA/fairchem, etc.).

git clone https://github.com/kamirian/Matpes_analysis.git
cd Matpes_analysis
pip install -r requirements.txt   # base requirements
# Then install your FP's additional package (e.g. pip install mace-torch)

Download the benchmark data

Pull the LFS-tracked files and download the large PBE/r2SCAN JSON files from the data repository (links in README). Place them in the data/ directory as specified in the notebooks.

Run inference with your model

Use the HPC job generator notebooks to produce per-atom force predictions for your model on the chosen dataset. The generator notebooks (generate/run_pbe.ipynb, generate/run_r2scan.ipynb, generate/run_omat24_rattled.ipynb) accept any ASE-compatible calculator. The key output is a JSON file with per-atom all_deltaF, all_deltaTheta, and all_F_dft_mags arrays for each structure.

# Example: add your model as a new key in all_results dict
all_results["MyModel"] = {
    "all_deltaF":      [...],   # |F_FP - F_DFT| per atom (eV/Å)
    "all_deltaTheta":  [...],   # angle between F_FP and F_DFT per atom (degrees)
    "all_F_dft_mags":  [...],   # |F_DFT| per atom (eV/Å)
}

Compute and report metrics

Run the analysis notebook (analysis/pbe_matpes.ipynb or analysis/r2scan_matpes.ipynb) with your model included in all_results. The notebook computes all leaderboard metrics automatically. Report the following columns:

RMSE |ΔF| (eV/Å)RMS |ΔF|, same filter↓ lower better

Analysis Code

Three Python modules contain all reusable analysis and visualization logic imported by the notebooks. They operate on plain Python dicts with arrays of per-atom values and are dataset-agnostic.

scripts/matpes_frac_analysis.py

Core analysis engine

The computational backbone of all three analysis notebooks. Provides:

  • Fraction tables — fraction (%) with |ΔF| below threshold, optionally joint with |Δθ|
  • Regime panels — near-equilibrium vs. far-from-equilibrium error statistics
  • Conditioned MAE/RMSE — error metrics as a function of |FDFT| or |Δθ| thresholds
  • Catastrophic-failure analysis — fraction with |ΔF| above threshold
  • Query utilities — find atoms meeting any combination of |ΔF|, |Δθ|, |FDFT| criteria
  • Visualization — all heatmap and histogram types
scripts/mlip_cdf_density_plots.py

CDF and density visualization

Handles all continuous distribution figures:

  • CDF computation — builds CDFs of |ΔF| and |Δθ| for all models, with near-eq / far-from-eq split
  • 2-D density panels — joint density heatmaps with marginal histograms
  • CDF plots with inset — full CDF with automatically placed zoom inset
  • PCHIP-smoothed CDF curves, log-norm density colormaps, per-model color and linestyle support
scripts/heatmap_table.py

Heatmap drawing primitives

Low-level matplotlib code for the custom split-triangle heatmap style — each cell is divided diagonally, with lower and upper triangles encoding two different statistics (e.g., |Δθ| < 1° vs < 20°, or MAE vs RMSE).

  • Cell-level: draw_triangular_cell, draw_rectangular_cell
  • Column-level: draw_triangular_column, draw_rectangular_column
  • Figure-level: triangular_heatmap_with_fraction_row, plot_fraction_panel
