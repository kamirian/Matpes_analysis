# MatPES MLIP Force Analysis

A systematic study of force prediction accuracy in machine learning interatomic potentials (MLIPs),
evaluated against DFT forces from the [MatPES](https://github.com/materialsvirtuallab/matpes) dataset.
The central argument: **total average error (MAE/RMSE) alone is an incomplete benchmark** because it
is diluted by the large fraction of near-equilibrium atoms, masking failure modes that matter in
realistic simulations.

---

## Motivation

Current MLIP evaluations often focus on total average error: MAE/RMSE of force magnitude |ŒîF| across
all atoms. While useful, these metrics only capture part of the picture. A model can achieve low
average error while still:

- Predicting forces with incorrect directions, leading to large force angle errors |ŒîŒ∏|
- Producing rare but severe large force-magnitude errors
- Performing well near equilibrium while failing on far-from-equilibrium (FE) configurations

These failure modes matter because force errors directly affect atomistic simulations. In molecular
dynamics, large or misdirected forces lead to unphysical atomic trajectories, unstable simulations,
or crashes. In static relaxations and NEB calculations, force accuracy requirements are strict, with
typical convergence thresholds of 0.001‚Äì0.01 eV/√Ö. The near-equilibrium majority of atoms in most
datasets dilutes these failures in the reported average, making a model appear more reliable than it is.

This work addresses that gap by evaluating force accuracy across three regimes:

| Regime | Definition | Why it matters |
|--------|-----------|----------------|
| Highly accurate & joint magnitude-angle accuracy | \|ŒîF\| < 0.01 eV/√Ö and \|ŒîŒ∏\| < 1‚Äì20¬∞ | Measures the fraction of predictions accurate enough for strict simulation settings; force convergence criteria in relaxations and NEB are typically 0.001‚Äì0.01 eV/√Ö |
| Large force-magnitude errors | \|ŒîF\| > 0.5‚Äì10 eV/√Ö | Rare but catastrophic: a single severely erroneous force during MD can destabilize a trajectory or cause simulation failure, yet never appears in a reported average |
| Far-from-equilibrium (FE) atoms | \|F_DFT\| ‚â• 1 eV/√Ö | Tests reliability on transition states, migration pathways, and rattled or strained structures ‚Äî configurations that matter most for NEB and high-temperature MD. FE atoms make up ~26.8% of MatPES vs ~6.1% of MPTrj |

---

## Metrics

To move beyond total average error, we evaluate five complementary metrics across all models:

| Metric | Definition | Why it matters |
|--------|-----------|----------------|
| Total average error | MAE/RMSE of \|ŒîF\| and \|ŒîŒ∏\| across all atoms | Baseline comparison; subject to dilution by the large fraction of near-equilibrium atoms |
| Highly accurate magnitude predictions | Fraction of atoms with \|ŒîF\| < 0.01 eV/√Ö | Directly relevant to force convergence criteria in static relaxations and NEB |
| Large force-magnitude errors | Fraction of atoms with \|ŒîF\| > 0.5‚Äì10 eV/√Ö | Rare but catastrophic; destabilizes MD trajectories and causes simulation failures |
| Far-from-equilibrium (FE) atoms | MAE/RMSE for atoms with \|F_DFT\| ‚â• 1 eV/√Ö (26.8% of MatPES) | Probes reliability on transition states and migration pathways, which are underrepresented in near-equilibrium datasets like MPTrj (~6.1%) |
| Joint magnitude-angle accuracy | Fraction of atoms with \|ŒîF\| < 0.01 eV/√Ö and \|ŒîŒ∏\| < 1¬∞ or 20¬∞ | Captures simultaneous accuracy in both force magnitude and direction; neither alone is sufficient |

## Analysis at a Glance

The notebooks produce a rich set of publication-quality figures:

- **Split-triangle heatmaps** ‚Äî fraction of atoms with |ŒîF| < threshold (lower triangle: also |ŒîŒ∏| < 1¬∞, upper: |ŒîŒ∏| < 20¬∞), showing joint magnitude-angle accuracy
- **Cumulative distribution functions** ‚Äî full CDF of |ŒîF| and |ŒîŒ∏| with zoomed inset on the high-accuracy tail
- **2-D density plots** ‚Äî joint distributions of |ŒîF| vs |ŒîŒ∏|, |ŒîF| vs F\_DFT, and |ŒîŒ∏| vs F\_DFT
- **Near-eq / far-from-eq regime panels** ‚Äî error statistics split by |F\_DFT| < 1 eV/√Ö vs ‚â• 1 eV/√Ö
- **Large-error fraction heatmaps** ‚Äî fraction of atoms with |ŒîF| > threshold (log-scale), the catastrophic failure metric
- **MAE/RMSE heatmaps** ‚Äî triangular heatmaps conditioned on |F\_DFT| thresholds (lower = MAE, upper = RMSE)
- **Error histograms** ‚Äî log-log |ŒîF| and linear-log |ŒîŒ∏| distributions, all atoms and near-equilibrium subset
- **Merged 6-panel overview figure** ‚Äî crystal structure + density plots + CDFs, ready for publication

## Python Modules

### `scripts/matpes_frac_analysis.py` ‚Äî Core analysis engine

The computational backbone of all three analysis notebooks. Provides:

- **Fraction tables** ‚Äî `build_frac_table`, `build_dF_frac_table`: compute the fraction of atoms with |ŒîF| below threshold, optionally also requiring |ŒîŒ∏| < cut
- **Regime panels** ‚Äî `build_regime_panels`, `build_theta_regime_panels`: split atoms into near-equilibrium (|F\_DFT| < 1 eV/√Ö) and far-from-equilibrium (|F\_DFT| ‚â• 1 eV/√Ö) and compute error statistics for each
- **Conditioned MAE/RMSE** ‚Äî `build_F_dft_conditioned_mae_rmse`, `build_theta_conditioned_mae_rmse`: error statistics as a function of |F\_DFT| or |ŒîŒ∏| thresholds
- **Large-error analysis** ‚Äî `build_dF_frac_larger_table`, `build_large_dF_fdft_table`: fraction of atoms with |ŒîF| *above* threshold and the |F\_DFT| distribution of those catastrophic-error atoms
- **Visualization** ‚Äî `split_triangle_heatmap`, `single_heatmap`, `plot_error_histograms`, `plot_fraction_panel`: all figure types used in the notebooks
- **Querying** ‚Äî `get_bad_atom_indices`, `get_bad_structure_indices`: find atoms/structures meeting any combination of |ŒîF|, |ŒîŒ∏|, and |F\_DFT| criteria

### `scripts/mlip_cdf_density_plots.py` ‚Äî CDF and density visualization

Handles the continuous distribution figures:

- **CDF computation** ‚Äî `build_cdf_from_all_results`, `build_cdf_by_regime`: compute cumulative distributions of |ŒîF| and |ŒîŒ∏| for all models, with optional near-eq / far-from-eq split
- **2-D density panels** ‚Äî `panel_abs_dF_vs_dtheta_cond_on_Fdft`, `panel_Fdft_vs_abs_dF`, `panel_Fdft_vs_dtheta`: joint density heatmaps with marginal histograms, PCHIP-smoothed CDFs
- **CDF plots** ‚Äî `plot_cdf_with_inset_on_ax`: full CDF with automatically placed inset zoom for the high-accuracy tail or model-separation region

### `scripts/heatmap_table.py` ‚Äî Heatmap drawing primitives

Low-level matplotlib code for the custom split-triangle heatmap style used throughout the paper. Provides `draw_triangular_cell`, `draw_rectangular_column`, `add_colorbar`, and the high-level `triangular_heatmap_with_fraction_row` that assembles a complete publication-style heatmap table.

## Dataset Chunking & Job Generators

The notebooks in `generate/` handle two things: (1) splitting a large dataset into chunks suitable for parallel HPC evaluation, and (2) generating self-contained SLURM job scripts for each MLIP √ó chunk combination. They are **not needed to reproduce the analysis** ‚Äî only to regenerate raw result JSON files or evaluate on a new dataset.

| Notebook | Dataset used | Output |
|----------|-------------|--------|
| `generate/run_pbe.ipynb` | MatPES-PBE (`.json.gz`) | `all_results_{mlip}_PBE.json` |
| `generate/run_r2scan.ipynb` | MatPES-r2SCAN (`.json.gz`) | `all_results_{mlip}_r2SCAN.json` |
| `generate/run_omat24_rattled.ipynb` | OMAT24 Rattled-1000 (`.tar.gz` / `.aselmdb`) | `all_results_{mlip}_PBE.json` |

### General-purpose pipeline ‚Äî works for any dataset

The chunking and job-generation pipeline is dataset-agnostic. It requires only that each structure has **atomic forces** and a **crystal structure** (positions + lattice). The workflow is:

```
1. Set CHUNK_DATASET_PATH, CHUNK_BASE_DIR, CHUNK_N in the notebook
2. Run the chunking cell ‚Üí writes chunk00/‚Ä¶/chunkNN as compressed JSON
3. Run the SLURM generator cell ‚Üí writes code_chunk_*/ directories + mass_submit.sh
4. rsync the generate/ output to your cluster
5. bash mass_submit.sh   (submits one job per MLIP √ó chunk)
6. Merge chunk outputs ‚Üí single all_results_{mlip}.json per model
```

#### Supported input formats

| Format | Extension | Notes |
|--------|-----------|-------|
| MatPES / generic JSON | `.json` | Must contain a list of entries with `forces`, `energy`, `structure` |
| Compressed JSON | `.json.gz` | Same schema, gzip-compressed |
| Extended XYZ | `.xyz` | Parsed with ASE; label inferred from filename |
| OMAT24 LMDB | `.tar.gz` / `.aselmdb` | Requires `ase-db-backends`; extracted automatically |

#### Key configuration parameters

```python
CHUNK_DATASET_PATH = "/path/to/your_dataset.xyz"   # input file
CHUNK_BASE_DIR     = "/path/to/chunks/"             # where chunks are written
CHUNK_N            = 20        # number of parallel chunks (= number of SLURM jobs per model)
CHUNK_FUNCTIONAL   = None      # label prefix for chunk filenames; None ‚Üí inferred from file/data
CHUNK_N_MAX        = None      # cap total entries (None = use all)
```

Set `CHUNK_N` to the number of parallel jobs you want per MLIP. Each chunk gets its own SLURM script. `CHUNK_FUNCTIONAL` is used as a filename prefix (e.g. `"pbe"`, `"r2scan"`, `"lyc"`); if `None`, it is inferred from the `functional` field in the data or from the filename.

### Example: LYC dataset

The LYC dataset (Li‚ÄìY‚ÄìCl solid electrolyte) is an `.xyz` file. To evaluate a fine-tuned MACE model on it:

```python
CHUNK_DATASET_PATH = "/path/to/full_dataset_corrected.xyz"
CHUNK_BASE_DIR     = "/path/to/LYC_chunks/"
CHUNK_N            = 5          # small dataset ‚Üí 5 chunks is enough
CHUNK_FUNCTIONAL   = None       # inferred as "full_dataset" from filename
CHUNK_N_MAX        = None
```

The resulting analysis (force errors of MACE-MP0 baseline vs. fine-tuned variants at different training sizes) is in `examples/lyc/analysis_lyc.ipynb`, with pre-computed results in `examples/lyc/results/`.
